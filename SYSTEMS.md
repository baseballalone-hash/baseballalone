## 1. 게임 흐름 개요

### 1.1 커리어 패스

```
고교 1학년(16세) ──→ 고교 2학년 ──→ 고교 3학년
                                        │
              ┌──────────────┬───────────┴──────────────┐
              │              │                          │
           대학교           KBO 드래프트             MLB 오퍼
        (1~4학년)          (신인 1~10라운드)       (마이너 → 콜업)
              │              │                          │
              └──────────────┴──────────────────────────┘
                                        │
                          프로 커리어 (pro2 ↔ pro1 ↔ mlb_a ↔ aa ↔ aaa ↔ mlb)
                                        │
                                      은퇴
                                        │
                              명예의 전당 판정
                                        │
                                회귀(NewGame+)
```

### 1.2 1주일 단위 진행

- **평일(월~금)**: 자동 훈련 1회 (방향 프리셋 기반 최부족 능력치 1종 상승)
- **주말**: 리그 경기 자동 시뮬레이션 (26주 시즌, 리그별 경기수 상이)
- **대회 주**: 토너먼트 결승 진출 시 라이브 모달 자동 발동 후 일시정지
- **시즌 종료**: 시즌 종합 화면 → 휴식기 카테고리 선택 → 다음 시즌

### 1.3 시즌 캘린더

| 리그 | 주수 | 주당 경기수 | 총 경기수 |
|---|---|---|---|
| 고교 | 26주 | 2경기 | 52 |
| 대학 | 22주 | 2경기 | 44 |
| KBO 1군 | 24주 | 6경기 | 144 |

## 2. 핵심 시스템

### 2.1 능력치 시스템

양방향(타자 + 투수) 능력치 체계. 타자/투수 역할 모두 동시 보유.

**타자 4종**

| 능력치 | 키 | 역할 |
|---|---|---|
| 컨택 | `contact` | 안타율 기반 |
| 파워 | `power` | 장타율/홈런 |
| 선구 | `eye` | 볼넷/삼진 |
| 주력 | `speed` | 도루/3루타 |

**투수 4종**

| 능력치 | 키 | 역할 |
|---|---|---|
| 구속 | `velocity` | 기본 피안타율 |
| 제구 | `control` | 볼넷/사구 억제 |
| 변화구 | `breaking` | 탈삼진 |
| 스태미나 | `stamina` | 투구 지구력·강판 임계 |

**stage별 캡 (상점 보너스 별도 +5씩 추가 가능)**

| 스테이지 | 기본 캡 |
|---|---|
| 고교 | 150 |
| 대학 | 175 |
| KBO 2군 | 200 |
| KBO 1군 | 250 |
| MLB 마이너 | 250~300 |
| MLB | 300 |

### 2.2 재능 시스템

캐릭터 생성 시 선택. 회귀 상점으로 최대 3슬롯까지 확장.

| Key | 이름 | 훈련/성장 가중 스탯 |
|---|---|---|
| `slugger` | 거포 | 파워 ×1.5, 컨택 ×1.3 |
| `sprinter` | 교타/주루 | 컨택 ×1.3, 주력 ×1.3, 선구 ×1.2 |
| `fireballer` | 강속구 | 구속 ×1.5, 스태미나 ×1.3 |
| `tactician` | 기교파 | 제구 ×1.5, 변화구 ×1.3 |
| `all_round` | 올라운더 | 전 스탯 ×1.1 |

### 2.3 자동훈련 프리셋

매일 자동으로 방향에서 가장 부족한 능력치 1종을 올림.

| Key | 이름 | 대상 스탯 |
|---|---|---|
| `slugger` | 거포 | 파워 집중 |
| `contact_speed` | 교타/주루 | 컨택·주력·선구 |
| `batter_balance` | 타자 밸런스 | 타자 4종 균등 |
| `fireballer` | 강속구 | 구속·스태미나 집중 |
| `breaking` | 변화구 | 변화구·제구 집중 |
| `pitcher_balance` | 투수 밸런스 | 투수 4종 균등 |
| `two_way` | 올 밸런스 | 전 스탯 균등 |

### 2.4 성장 곡선

```
나이별 훈련 효율 (ageMultiplier):
  16~18세: ×1.6
  19~22세: ×1.4
  23~25세: ×1.2
  26~29세: ×1.0
  30~32세: ×0.6
  33~35세: ×0.4
  36~37세: ×0.15
  38세+  : ×0.05
```

**노화 감쇄** (시즌 종료 시 확률적 stat 감소):

| 나이 | 발동 확률 | 감소량 |
|---|---|---|
| 30~32세 | 10% | -1 |
| 33~35세 | 30% | -3 |
| 36~37세 | 50% | -4 |
| 38~39세 | 70% | -5 |
| 40세+ | 85% | -6 |

### 2.5 경기 시뮬레이션

**매치업 공식**: `softDiff` 격차 평탄화 → K/BB/HR/in-play 분기

- **OVR 중앙화**: `ovrHelper.js` — `getBatterOVR` / `getPitcherOVR` / `getCombinedOVR`
- **NPC-only 고속 패스**: 플레이어 미참여 경기는 타석 이벤트 생략, OVR 기반 점수 직접 산출
- **코치판단**: 메인 투수 OVR vs 팀 SP 평균 비율로 등판 확률 결정
- **구종 시스템**: 5구종 (직구/슬라이더/커브/체인지업/포크), 좌우 매치업 페널티
- **스태미나 피로**: 등판 후 타자수 누적 초과 시 구위/제구 감쇄
- **멘탈 클러치**: 고압 상황(7회+ + 접전 + 득점권)에서 멘탈 능력치 영향

### 2.6 부상 시스템

| 구분 | 회복 | 발동 조건 |
|---|---|---|
| 경상 (minor) | 1주 | 훈련/HBP 2% 굴림 |
| 중상 (moderate) | 3주 | — |
| 중증 (severe) | 8주 | — |
| 토미존 수술 | 30주 + 영구 페널티 | severe + 어깨/팔꿈치 + 30% 추가 굴림 |

부상 부위 7종: 어깨 / 팔꿈치 / 무릎 / 발목 / 햄스트링 / 허리 / 손목

### 2.7 커리어 시스템

| 시스템 | 내용 |
|---|---|
| 진로 분기 | 고교 졸업 시 대학 / KBO 드래프트 / MLB 오퍼 / 은퇴 선택 |
| KBO → MLB | 9시즌+ 해외 FA / 7~8시즌 포스팅 제도 |
| FA | 4년 계약 만료 → 잔류/이적 모달 |
| 트레이드 | 휴식기 진입 시 8% 확률 NPC 제안 (명성 50 미만은 자동 이적) |
| 콜업/강등 | 능력치 임계 기반 자동 판정 + 모달 |
| 군 입대 | pro1/pro2 + 만 27세 → 상무/경찰청/사회복무 선택 (한국어 모드) |
| 병역 면제 | 올림픽 동메달+, 아시안게임 금메달 → 자동 면제 |
| 강제 은퇴 | 최하위 리그 평균 이하 또는 만 41세+ → 방출 |

### 2.8 회귀 시스템 (NewGame+)

은퇴 시 명예의 전당 점수가 회귀 잔액으로 적립. 5탭 상점에서 영구/캐릭터별 강화.

| 탭 | 내용 |
|---|---|
| 재능 슬롯 | 최대 3슬롯까지 확장 (영구) |
| Stage 캡 | 스탯별 캡 +5 (영구, 상한 없음) |
| 시작 능력치 | 캐릭터 생성 초기값 상승 (캐릭터당 1회) |
| 특성 | 12종 특성 효과 (캐릭터당 1회) |
| 유물 | 레벨업 가능 강화 아이템 (캐릭터당, 재구매 시 레벨업) |

**도전과제 4종**: `walkoff_one` / `championship_one` / `hof_inducted` / `severe_recovered`

## 4. i18n 아키텍처

**핵심 규칙**: 시스템 레이어는 키만, UI 레이어가 `t()`로 라벨 해결.

| 레이어 | 정책 |
|---|---|
| `src/i18n/{ko,en}.js` | 도메인별 네임스페이스의 플랫한 문자열 테이블. 키는 dot-notation (`stat.contact`, `preset.slugger.label`) |
| `src/i18n/index.js` | `t(key, params)`, `setLocale`, `toggleLocale`, `formatGameDate` |
| `src/systems/*` | 라벨/문자열 보유 금지. 데이터 객체는 키만 보유. 로그는 `t()` 즉시 렌더 후 텍스트로 동결 |
| `src/data/*` | 이름/팀 풀은 locale별 파일 분리. `getTeamPool(stage, locale)`, `randomName(locale)` 디스패치 |
| `src/views/*` | 모든 표시 문자열은 `t()` 호출. 토글 시 `route(state.view)` 재호출로 즉시 재렌더 |

### 데이터 "동결" vs "동적" 분류

| 범주 | 정책 | 이유 |
|---|---|---|
| UI 라벨, 버튼 | **동적** (매 렌더 `t()`) | 토글 즉시 반영 |
| 캐릭터 이름, 팀 이름 | **동결** (생성 시점 locale 풀에서 결정) | 식별자 성격 — 토글로 바뀌면 혼란 |
| 로그 메시지 | **동결** (작성 시점 `t()` 결과를 문자열 저장) | 시간순 기록 정합성 |
| 부상/재능/훈련 enum 표시 | **동적** (key 저장 → 표시 시 `t()`) | 현재 언어로 보여야 함 |
| 날짜 포맷 | **동적** (`formatGameDate(d)`) | `3월 15일 (수)` ↔ `Mar 15 (Wed)` |

### 새 언어 추가

1. `src/i18n/<locale>.js` 작성 — `ko.js` 구조 그대로 모든 키 채움 (누락 시 `DEFAULT_LOCALE` 폴백)
2. `src/i18n/index.js`의 `SUPPORTED_LOCALES`, `MONTH_ABBR`, `TABLES`에 추가
3. (선택) 이름/팀 풀 필요 시 `src/data/names.<locale>.js`, `src/data/teams.<locale>.js` 추가
4. 3개 이상 언어 지원 시 `toggleLocale()`을 셀렉터로 교체
